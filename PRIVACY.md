# VoteCalc 隐私政策

最后更新：2026-06-14

本隐私政策适用于 VoteCalc Discord 机器人（以下简称“VoteCalc”、“本机器人”或“本项目”）。VoteCalc 是一个用于 Discord 赛事、作品评价、论坛帖子反应统计的工具，主要功能包括读取指定 Discord 帖子或频道链接、统计反应、筛选有效评价用户、按角色或分区权重计算结果，并向命令调用者或服务器频道输出统计结果。

## 1. 我们收集哪些数据

VoteCalc 仅在执行统计功能所必需的范围内处理 Discord 数据。根据你使用的命令参数和服务器配置，VoteCalc 可能会临时访问或处理以下信息：

### 1.1 命令调用信息

当用户使用 `/analyze` 等命令时，VoteCalc 可能会处理：

- 调用命令的 Discord 用户 ID、用户名、服务器昵称；
- 命令参数，例如：
  - 待统计的 Discord 链接；
  - 最少有效评价数量；
  - 是否输出有效用户；
  - 是否公开输出结果；
  - 有效反应表情列表；
  - 角色权重配置；
  - 分区权重配置。

### 1.2 帖子、频道与反应数据

为了完成投票统计，VoteCalc 可能会读取：

- 用户在命令中提供的 Discord 帖子、线程或频道链接；
- 相关帖子或线程的名称、ID、所属频道或服务器信息；
- 相关消息上的反应表情；
- 每个反应表情的数量；
- 对最高反应或指定有效反应作出反应的用户列表；
- 分组作品所包含的多个帖子或线程之间的关联关系。

### 1.3 用户与角色数据

如果启用了有效用户筛选、角色权重或有效用户列表输出，VoteCalc 可能会临时处理：

- Discord 用户 ID；
- Discord 用户名；
- 用户在本Bot所能访问的服务器中的显示名；
- 用户在本Bot所能访问的服务器中的角色名称；
- 用户角色对应的权重；
- 角色权重分布统计；
- 某个用户是否属于有效评价用户。

### 1.4 日志数据

VoteCalc 运行时可能会向标准输出、systemd journal、Docker logs 或其他部署环境日志系统写入调试信息，包括但不限于：

- 命令调用者用户名；
- 输入的 Discord 链接；
- 被解析的频道 ID、消息 ID；
- 统计到的反应数量；
- 有效用户数量；
- 角色匹配情况；
- 错误信息与异常信息。

这些日志主要用于排查部署、权限、Discord API 调用和统计过程中的问题。

## 2. 我们不主动收集哪些数据

VoteCalc 的设计目的不是聊天记录归档、用户画像、广告投放或行为追踪。因此，本机器人默认不主动收集以下信息：

- 用户的私信内容；
- 用户的语音、视频、屏幕共享内容；
- 用户的邮箱、电话号码、真实姓名、支付信息；
- Discord 之外的网站访问记录；
- 与投票统计无关的频道历史消息；
- 附件、图片、文件内容，除非这些内容本身出现在被统计消息或嵌入内容中，并且统计流程需要读取其中的 Discord 链接。

需要注意的是，如果赛事主办者将包含 Discord 链接的汇总消息提供给 VoteCalc，VoteCalc 可能会读取该消息正文或嵌入描述中的链接，用于进一步解析参赛帖子。

## 3. 我们如何使用数据

VoteCalc 使用上述数据仅用于以下目的：

1. 解析赛事主办者或授权用户提供的 Discord 链接；
2. 获取指定帖子、线程或频道上的反应数据；
3. 识别每个作品的最高反应或指定有效反应；
4. 根据最少有效评价数量筛选有效用户；
5. 根据角色权重或分区权重计算加权分数；
6. 生成赛事统计结果、排名、有效评价数量、比例和角色权重分布；
7. 在用户明确选择输出有效用户时，输出有效用户列表；
8. 检查命令调用者是否在白名单内或是否拥有允许使用机器人的角色；
9. 记录必要的调试日志，以便维护者排查错误和改进稳定性。

VoteCalc 不会将收集到的数据用于广告、用户画像、出售数据、跨服务器商业追踪，或与投票统计无关的用途。

## 4. 数据是否会公开

VoteCalc 的统计结果可能根据命令参数以不同方式显示：

- 默认情况下，统计结果可以配置为仅命令调用者可见；
- 如果命令调用者启用了公开输出，统计结果可能会显示在 Discord 频道中，频道内有权限查看该频道的成员都可能看到；
- 如果启用了有效用户列表输出，结果中可能包含有效用户的用户名、服务器昵称和角色身份组信息；
- 统计结果可能包含帖子标题、帖子链接、最高反应、总反应数、有效评价人数、加权分数、比例和角色权重分布。

## 5. 数据存储与保留

VoteCalc 当前主要在命令执行过程中临时处理数据。除非部署者额外修改代码或配置外，VoteCalc 本身不以数据库形式长期保存投票明细。

但是，以下位置可能会保留部分数据：

- Discord 本身保存的原始帖子、反应、用户信息和 Bot 输出消息；
- 运行 VoteCalc 的服务器日志；
- systemd journal、Docker logs、云服务日志或其他部署平台日志；


## 6. 数据共享

VoteCalc 不会主动出售、出租或商业化共享用户数据。

VoteCalc 可能在以下情况下涉及数据传输或披露：

1. **Discord API**：VoteCalc 通过 Discord API 读取服务器、频道、帖子、反应和成员角色信息。Discord 对其自身服务中的数据处理适用 Discord 的隐私政策和服务条款。
2. **部署环境服务商**：如果 VoteCalc 部署在云服务器、容器平台或日志平台上，相关服务商可能会根据部署者的配置处理日志或运行数据。
3. **服务器管理员或赛事主办者**：当统计结果被公开输出、复制、导出或用于赛事排名时，相关结果可能被服务器管理员、赛事主办者或频道成员看到。
4. **法律要求**：如果适用法律、法院命令或监管要求要求披露，部署者可能需要依法处理相关请求。

除上述情况外，VoteCalc 不会主动向第三方共享 Discord API 数据。

## 7. 数据安全

VoteCalc 的维护者和部署者、使用者应采取了合理的安全措施保护数据，包括：

- 妥善保管 Discord Bot Token；
- 不将 `appsettings.json` 中的 Token、代理地址、服务器 ID、白名单用户 ID 等敏感配置提交到公开仓库；
- 限制服务器日志访问权限；
- 限制可使用 `/analyze` 命令的白名单用户和角色；
- 定期更新依赖和运行环境；
- 在不需要公开结果时使用非公开响应；
- 避免在不必要时启用有效用户列表输出。

## 8. 用户的数据访问、删除与更正请求

如果你希望访问、删除或更正 VoteCalc 处理过的与你相关的数据，可以联系 VoteCalc 的对应部署者或服务器管理员。

可请求的事项包括：

- 删除或编辑 Bot 已经公开发送的统计结果消息；
- 删除部署服务器日志中与你相关的条目；
- 停止在后续统计结果中公开显示有效用户列表；
- 更正错误的角色权重配置或统计配置；
- 询问你的数据是否被某个 VoteCalc 实例处理过。

需要注意：

- VoteCalc 无法删除 Discord 平台本身保存的原始反应、帖子、服务器昵称或角色信息；
- 如果统计结果已经被其他用户截图、复制、转发或重新发布，VoteCalc 无法控制这些副本；
- 删除 Bot 输出消息可能需要服务器管理员权限或 Bot 对应频道权限。

联系方式：

- GitHub Issues：`https://github.com/MagicalAstrogy/VoteCalc/issues`

## 9. 儿童与未成年人

VoteCalc 面向 Discord 服务器中的赛事统计和社区管理场景，不专门面向儿童提供服务。

部署者和服务器管理员应确保其服务器、赛事和机器人使用方式符合 Discord 的年龄要求、社区规则以及适用地区的未成年人保护要求。如果服务器中存在未成年用户，建议避免公开输出包含个人用户列表、角色身份或其他可能造成识别风险的信息。

- GitHub Issues：`https://github.com/MagicalAstrogy/VoteCalc/issues`
